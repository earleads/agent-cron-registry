Get an always-on server

The main setup needs a computer that never sleeps, so your scheduled jobs actually run at 3am when your laptop is shut. Here's how to rent a small one for a few euros a month and lock it down properly, in about ten minutes. No prior server experience needed.

Start the walkthrough Main setup guide

Why you need this

A scheduled job only runs if the computer it lives on is awake. Your own laptop sleeps, closes, and travels with you, so any job on it goes quiet the moment the lid shuts. The fix is a small "always-on" computer in a data centre: it stays powered on around the clock, so your Monday 9am report actually goes out on Monday at 9am. These are cheap, rentable by the month, and you never see the physical machine, you just connect to it over the internet.

This walkthrough uses Hetzner because it's one of the cheapest and simplest providers, but the same idea works on any provider (DigitalOcean, Linode, AWS Lightsail). If you already have an always-on Mac or Linux box somewhere, you can skip all of this and use that instead.

Before you start

  • A Hetzner account. Sign up free at console.hetzner.com. You'll add a card, but you're only charged for what you run, and the smallest server is a few euros a month.
  • An SSH key. This is the digital "key" that lets you (and only you) into the server, instead of a password. Don't worry if you don't have one, step 1 makes it, and your agent can do it for you.
  • Your coding agent (like Claude Code), same as the main guide. It can do most of the connecting and locking-down for you if you'd rather not touch a terminal.
  • About ten minutes.

How to use this guide

Server setup needs a bit more terminal than the main guide, because you're logging into a brand-new machine that doesn't have your agent on it yet. So a few commands here are copy-paste. Where your agent can do the work for you, there's a "Say to your agent" line as usual. Every command comes with a plain-English explanation of what it's doing, so nothing is a mystery.

Setup, step by step

1

Make your SSH key

This is the private key that unlocks the server. You make it once on your own computer. If you've ever connected to GitHub over SSH you probably already have one, and your agent can just reuse it.

Say to your agent "Check if I already have an SSH key, and if not create one. Then show me the public key so I can paste it into Hetzner."
Run it yourself 
# create a key (press Enter through the prompts):
ssh-keygen -t ed25519

# then print the PUBLIC half to copy (safe to share):
cat ~/.ssh/id_ed25519.pub

Copy the whole line it prints (it starts with ssh-ed25519). That's the public key, the only half you ever paste anywhere. The other file, without .pub, is private, never share it.

2

Create the server in Hetzner

You do this one

This happens in Hetzner's website, so it's on you. It takes about two minutes.

  1. Go to console.hetzner.com and open (or create) a project.
  2. Click Add Server.
  3. Location: pick whichever is closest to you (any is fine).
  4. Image: choose Ubuntu (the latest LTS version listed).
  5. Type: the smallest shared instance is plenty to start. If you expect to run a lot of jobs or an always-on agent, pick a slightly roomier one, you can resize later without losing anything.
  6. SSH keys: click "Add SSH key" and paste in the public key from step 1. This is what lets you in.
  7. Backups: tick this on. It's a small extra cost and means a bad change is never permanent.
  8. Give it a name (anything, e.g. jobs-server) and click Create & Buy now.

When it finishes, Hetzner shows you the server's IP address, a string of numbers like 203.0.113.10. That's its address on the internet. Keep it handy for the next step.

3

Connect to it for the first time

Now you reach into the new machine. The very first login is as the built-in root (admin) account.

Say to your agent "Connect to my new server as root at this address: [paste the IP from step 2]."
Run it yourself 
ssh root@<your-server-ip>

The first time, it asks if you trust this machine, type yes. Because you added your key in step 2, it lets you straight in with no password.

4

Lock it down

A fresh server on the open internet gets probed by bots within minutes, so before anything else you harden it: make a normal (non-root) user for daily use, turn on a firewall, add a tool that auto-bans repeated break-in attempts, and switch on automatic security updates. This is a one-time thing.

Say to your agent "Harden this fresh Ubuntu server for me: create a sudo user called [your name], copy my SSH key over to it, turn on the firewall allowing only SSH, and enable fail2ban and automatic security updates."
Run it yourself 
# update the system first
apt update && apt upgrade -y

# make a normal user for yourself (replace 'me')
adduser me
usermod -aG sudo me

# give that user your SSH key so you can log in as them
mkdir -p /home/me/.ssh
cp ~/.ssh/authorized_keys /home/me/.ssh/
chown -R me:me /home/me/.ssh
chmod 700 /home/me/.ssh
chmod 600 /home/me/.ssh/authorized_keys

# firewall (SSH only), auto-ban tool, auto security updates
apt install -y ufw fail2ban unattended-upgrades
ufw allow OpenSSH
ufw --force enable
dpkg-reconfigure -plow unattended-upgrades

What each part does, in plain English: the first block updates the software; the second makes a day-to-day account so you're not living as the all-powerful root; the third copies your key onto that account; the last installs a firewall that only allows the connection you use, a guard that blocks repeat intruders, and hands-off security patches.

5

Log back in as your normal user

From now on you connect as the user you just made, not as root. Test it works before you close the root session, so you're never locked out.

Say to your agent "Now connect to the server as [your name] to confirm it works, then we're done with root."
Run it yourself 
ssh me@<your-server-ip>
That's it, you now have a small computer that's always on and locked down. This is the machine the main guide's jobs run on. Head back and pick up from step 1: the main setup guide.

Good to know

  • Cost. The smallest instance runs a few euros a month; a roomier one with backups is closer to ten. You only pay while it exists, delete it in the Hetzner console and billing stops.
  • Outgrew it? In the Hetzner console you can resize a server to more power in a couple of clicks. It keeps the same address and all its data.
  • Locked out? If you ever lose your key, you're not stuck: the Hetzner console has a browser-based terminal and a "rescue mode" to get back in and add a new key. Worst case is a few minutes, not a lost server.
  • Other providers. DigitalOcean, Linode, and AWS Lightsail work the same way, create an Ubuntu box, add your SSH key, then run the same step 4 to lock it down.
